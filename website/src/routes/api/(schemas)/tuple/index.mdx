# tuple

Creates a tuple schema.

```ts
// Tuple schema with an optional pipe
const Schema = tuple<TItems>(items, pipe);

// Tuple schema with an optional message and pipe
const Schema = tuple<TItems>(items, message, pipe);

// Tuple schema with rest and optional pipe
const Schema = tuple<TItems, TRest>(items, rest, pipe);

// Tuple schema with rest and optional message and pipe
const Schema = tuple<TItems, TRest>(items, rest, message, pipe);
```

## Generics

- `TItems` <Property {...properties.TItems} />
- `TRest` <Property {...properties.TRest} />

## Parameters

- `items` <Property {...properties.items} />
- `rest` <Property {...properties.rest} />
- `message` <Property {...properties.message} />
- `pipe` <Property {...properties.pipe}/>

### Explanation

With `tuple` you can validate the data type of the input and whether the content matches `items` and `rest`. With `pipe` you can transform and validate the further details of the tuple. If the input is not an array, you can use `message` to customize the error message.

> By default, `tuple` removes unknown items from the input. You can change and control this behavior with `rest`. Read more <Link href="/guides/arrays/#rest-argument">here</Link>.

## Returns

- `Schema` <Property {...properties.Schema} />

## Examples

The following examples show how `tuple` can be used. Please see the <Link href="/guides/arrays/">arrays guide</Link> for more examples and explanations.

### Simple tuple schema

Schema to validate a tuple with two items.

```ts
const SimpleTupleSchema = tuple([string(), number()]);
```

### Tuple schema with rest

Schema to validate a tuple with generic rest items.

```ts
const TupleSchemaWithRest = tuple([string(), number()], boolean());
```

### Strict tuple schema

Schema to validate a strict tuple that prevent unknown items.

```ts
const StrictTupleSchema = tuple([string(), number()], never());
```
