# set

Creates a set schema.

```ts
// Set schema with an optional pipe
const Schema = set<TValue>(value, pipe);

// Set schema with an optional message and pipe
const Schema = set<TValue>(value, message, pipe);
```

## Generics

- `TValue` <Property {...properties.TValue} />

## Parameters

- `value` <Property {...properties.value} />
- `message` <Property {...properties.message} />
- `pipe` <Property {...properties.pipe} />

### Explanation

With `set` you can validate the data type of the input and whether the content matches `value`. With `pipe` you can transform and validate the further details of the set. If the input is not a set, you can use `message` to customize the error message.

## Returns

- `Schema` <Property {...properties.Schema} />

## Examples

The following examples show how `set` can be used.

### String set schema

Schema to validate a set with string values.

```ts
const StringSetSchema = set(string());
```

### Object set schema

Schema to validate a set with object values.

```ts
const ObjectSetSchema = set(object({ key: string() }));
```
